# Hrisa Docs - Visual Identity Design Briefs

## Design Concepts Overview

**App Name**: Hrisa Docs
**Tagline**: Document Synthesis powered by Tunisian Spirit
**Core Identity**: Professional document processing with bold Tunisian cultural pride

## Direction A: "Spicy Documents"

### Concept
Fusion of document/paper imagery with Tunisian chili pepper, emphasizing the "heat" of intelligent processing.

### Visual Description

**Primary Icon Design**:
- **Base**: Clean white document/paper sheet (folded corner, portrait orientation)
- **Accent**: Bold red Tunisian chili pepper (elongated, curved) emerging from or crossing the document
- **Style**: Flat design, minimal gradients, sharp clean lines
- **Composition**:
  - Document fills 70% of icon space
  - Chili pepper diagonal from bottom-left to top-right
  - Slight shadow/depth on paper fold
  - Chili pepper glossy red (#C62828)

**Variations**:
1. **Version A1**: Single red chili crossing white document diagonally
2. **Version A2**: Three chili peppers forming a bookmark on document edge
3. **Version A3**: Document stack with red chili as "spine" or binding element
4. **Version A4**: Circular icon - document pages radiating around central chili pepper

### Color Specifications
- **Primary Red**: #C62828 (chili pepper body)
- **Highlight Red**: #D32F2F (chili pepper highlights)
- **Shadow Red**: #B71C1C (chili pepper shadows)
- **Document White**: #FAFAFA (paper)
- **Document Shadow**: #E0E0E0 (paper fold/shadow)
- **Outline/Text**: #212121 (if needed)

### Typography (for logo with text)
- **Primary**: Montserrat Bold or Poppins SemiBold
- **Weight**: 600-700
- **Style**: Modern, clean, professional
- **Hierarchy**: "Hrisa" larger/bold, "Docs" smaller/lighter

## Direction B: "Tunisian Scholar"

### Concept
Combines Tunisian Chechia (traditional red hat) with academic/document elements, representing cultural pride meets intellectual pursuit.

### Visual Description

**Primary Icon Design**:
- **Base**: Traditional red Chechia (Tunisian fez) silhouette
- **Integration**: Document scroll, book pages, or paper emerging from hat
- **Style**: Clean iconography with cultural authenticity
- **Composition**:
  - Chechia as main shape (60% of space)
  - Document/scroll elements flowing from top or side
  - Optional: Subtle tassel detail from Chechia
  - Gold accent for tassel (#D4AF37)

**Variations**:
1. **Version B1**: Red Chechia with document scroll emerging from top
2. **Version B2**: Side profile of Chechia with stacked papers inside silhouette
3. **Version B3**: Circular badge - Chechia centered with document border
4. **Version B4**: Abstract geometric Chechia shape filled with document texture

### Color Specifications
- **Chechia Red**: #C62828 (main hat body)
- **Chechia Dark**: #B71C1C (hat shadows/depth)
- **Tassel Gold**: #D4AF37 (accent detail)
- **Document White**: #FAFAFA (papers)
- **Outline**: #212121 (defining edges)

### Cultural Notes
- Chechia should be authentic Tunisian style (cylindrical, flat top)
- Not Turkish fez style (tapered)
- Consider traditional embroidery patterns (subtle)

## Icon Testing Checklist

When you have concepts, test them at these sizes:
- [ ] 1024x1024 (source)
- [ ] 512x512 (retina displays)
- [ ] 256x256 (standard)
- [ ] 128x128 (Dock)
- [ ] 64x64 (toolbar)
- [ ] 32x32 (list view)
- [ ] 16x16 (smallest - must still be recognizable)

### Recognition Test
Ask yourself at each size:
1. Can I identify the main shape immediately?
2. Is it distinguishable from other apps?
3. Do the colors still work?
4. Are there too many details (causing blur/noise)?

## Next Steps

### Phase 1: Generate Concepts (Do This First)
1. Use AI prompts above in DALL-E, Midjourney, or Stable Diffusion
2. Generate 2-3 variations per direction = 15-20 initial concepts
3. Save all outputs to `design/concepts/`

### Phase 2: Review & Select
1. Review all concepts together
2. Select top 5 favorites
3. Note what works and what doesn't

### Phase 3: Refine
1. Take top concepts into vector software (Figma/Inkscape)
2. Clean up, adjust colors, perfect shapes
3. Export at all required sizes

### Phase 4: Test
1. Set as actual app icon
2. View on actual device/desktop
3. Check at all sizes
4. Get feedback from alpha testers
